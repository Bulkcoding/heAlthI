# STRONGER React 기술 선택

## 1. 결론

이 서비스는 `React 기반`으로 가고, 구체적인 프레임워크는 `Next.js`를 사용합니다.

즉 구조는 아래처럼 정리합니다.

- UI 프레임워크: React
- 앱 프레임워크: Next.js
- 언어: TypeScript
- 스타일링: CSS 변수 기반 글로벌 스타일 + 컴포넌트 분리
- 추후 확장: Tailwind, UI 라이브러리, 서버 액션, 백그라운드 작업

## 2. 왜 React 단독 SPA가 아니라 Next.js인가

이 서비스는 단순 소개 사이트가 아니라 아래 기능이 동시에 필요합니다.

- 회원가입 / 로그인
- 운동 기록 저장
- 추천 데이터 조회
- 관리자 화면
- 추후 SEO가 필요한 랜딩 페이지
- API 라우트 또는 서버 기능

이 요구사항을 보면 `React 단독 SPA`보다 `React 기반 Next.js`가 훨씬 현실적입니다.

## 3. 실제 개발 기준

첫 버전은 아래 기준으로 구현합니다.

- App Router 기반
- 반응형 웹 우선
- 서버 연동 전까지는 목업 데이터로 화면 구현
- 이후 Prisma / PostgreSQL 연결
- 추천 엔진과 분석 로직은 API 또는 서버 레이어로 분리

## 4. 바로 다음 단계

React 기반이라고 방향이 정리됐기 때문에, 이제부터는 아래 순서로 개발합니다.

1. Next.js 앱 골격 생성
2. 공통 레이아웃 생성
3. 로그인 / 대시보드 / 운동 기록 화면 구현
4. DB 연결
5. 추천 / 분석 로직 연결

이 문서를 기준으로 이후 작업은 모두 `React + Next.js` 전제로 진행합니다.
