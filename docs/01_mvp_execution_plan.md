# STRONGER MVP 실행 계획

## 1. 지금 가장 먼저 해야 할 일

이 서비스는 기능 욕심을 내면 너무 쉽게 커집니다. 그래서 첫 단계에서는 "모든 기능을 넣는 것"이 아니라 "유저가 실제로 운동을 기록하고, 다음 운동 추천을 믿고 다시 돌아오게 만드는 것"에 집중해야 합니다.

가장 먼저 해야 할 일은 아래 3가지를 고정하는 것입니다.

1. MVP 범위를 확정한다.
2. 기술 스택과 개발 구조를 확정한다.
3. 첫 구현 순서를 고정한다.

이 세 가지가 흔들리면, 화면을 먼저 만들더라도 나중에 대부분 다시 뜯어고치게 됩니다.

## 2. 내가 기준으로 잡은 MVP 범위

### 포함
- 회원가입 / 로그인
- 온보딩
- 운동 기록 저장
- 운동 세션 상세 보기
- 대시보드
- 다음 운동 세팅 추천
- 기본 운동 분석
- 운동 시간 기반 루틴 조정
- 친구 추가 및 활동 피드의 최소 버전
- 성취 배지의 최소 버전

### 제외
- 영상 자세 분석
- 웨어러블 연동
- 결제 고도화
- 대규모 챌린지 운영 기능
- 실시간 채팅
- 트레이너 전용 B2B 기능
- 고도화된 랭킹 시즌 시스템

이렇게 잘라야 1차 버전을 실제로 출시 가능한 수준까지 가져갈 수 있습니다.

## 3. 내가 추천하는 개발 우선순위

### Step 1. 제품 뼈대 만들기
목표는 "회원이 로그인해서 기본 구조를 둘러볼 수 있는 상태"입니다.

포함 항목:
- 프로젝트 초기 세팅
- 디자인 토큰 정의
- 반응형 레이아웃 프레임
- 로그인 / 회원가입
- 기본 대시보드 골격
- 좌측 네비게이션 / 모바일 탭 구조

완료 기준:
- 웹에서 주요 화면 이동 가능
- 모바일 / 태블릿 / PC 레이아웃이 최소한 깨지지 않음

### Step 2. 운동 기록 기능 만들기
목표는 "유저가 실제로 한 번의 운동을 끝까지 기록할 수 있는 상태"입니다.

포함 항목:
- 운동 시작
- 세트별 중량 / 횟수 / 성공 여부 입력
- 세션 종료
- 운동 기록 저장
- 최근 기록 조회
- 세션 요약 화면

완료 기준:
- 한 세션의 운동 데이터가 끊기지 않고 저장됨
- 이전 기록과 현재 기록을 비교해 볼 수 있음

### Step 3. AI 추천의 1차 버전 만들기
목표는 "기록을 기반으로 다음 운동 추천을 생성하는 상태"입니다.

포함 항목:
- 규칙 기반 추천 엔진
- 다음 운동 중량 / 세트 / 횟수 추천
- 운동 시간에 따라 루틴 길이 조정
- 실패 누적 / 과피로 시 감량 로직
- 추천 이유 설명 문구

완료 기준:
- 추천값이 무작위가 아니라 기록과 규칙에 근거해 생성됨
- 유저가 추천을 수락하거나 수정할 수 있음

### Step 4. 분석 기능 만들기
목표는 "유저가 자신의 운동 추세를 이해할 수 있는 상태"입니다.

포함 항목:
- 총 볼륨
- 운동별 최고 기록
- 주간 운동 빈도
- 예상 1RM
- 수행 점수
- 간단한 개선 피드백

완료 기준:
- 대시보드와 기록 상세 페이지에서 추세 확인 가능
- 세션별로 잘한 점 / 개선점 / 다음 액션이 노출됨

### Step 5. 소셜 최소 기능 만들기
목표는 "기록이 동기부여로 이어지는 상태"입니다.

포함 항목:
- 친구 추가
- 친구 활동 피드
- PR 공유
- 간단한 응원
- 기본 배지

완료 기준:
- 운동 완료 이벤트가 피드에 반영됨
- 공개 범위 설정이 동작함

### Step 6. 장비 인식 베타 만들기
목표는 "사진 인식이 실제 운동 기록으로 연결되는 상태"입니다.

포함 항목:
- 사진 업로드
- 장비 분류
- 장비에 연결된 운동 목록 노출
- 추천 운동 진입 버튼

완료 기준:
- 인식 결과를 사용자가 확인하고 수정 가능
- 오인식이 나와도 기록 흐름이 깨지지 않음

## 4. 실제 개발 순서

아래 순서로 가는 것이 가장 현실적입니다.

1. `기획 문서 고정`
2. `디자인 시스템 / 화면 구조 정의`
3. `DB 모델 설계`
4. `인증 / 회원 시스템`
5. `운동 기록 기능`
6. `분석 집계 로직`
7. `AI 추천 엔진`
8. `소셜 기능`
9. `장비 인식`
10. `운영자 기능`
11. `테스트 / 배포`

핵심은 `AI보다 기록 기능이 먼저`라는 점입니다. 기록 데이터가 안정적이지 않으면 추천도 분석도 모두 부정확해집니다.

## 5. 기술 스택 제안

현실적인 1차 빌드를 기준으로 아래처럼 가는 것을 추천합니다.

### 프론트엔드
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui 또는 최소한의 커스텀 컴포넌트 시스템
- React Query
- Zustand 또는 Context 기반 UI 상태관리

### 백엔드
- Next.js API Route 또는 Route Handler로 시작
- PostgreSQL
- Prisma ORM
- Redis

### AI / 분석
- 규칙 기반 추천 엔진
- 백그라운드 집계 작업
- 추후 LLM 설명 레이어 추가
- 추후 이미지 인식 모델 연동

### 인프라
- Vercel 또는 웹 프론트 배포 플랫폼
- Supabase / Neon / RDS 계열 DB
- S3 호환 이미지 저장소
- 로그 및 에러 모니터링

## 6. 네가 지금 해야 하는 일

사실 지금은 사용자인 네가 많이 할 일은 없습니다. 내가 기준을 잡아서 진행하면 됩니다.

그래도 정말 한 가지를 꼽으면 이것입니다.

- "첫 버전은 AI 만능 코치가 아니라, 기록과 추천이 믿을 만한 서비스"라는 방향만 유지하면 됩니다.

즉, 지금 당장 네가 해야 할 것은 기능을 더 늘리는 것이 아니라, MVP 범위를 흔들지 않는 것입니다.

## 7. 내가 바로 이어서 만들 순서

이 문서 다음 단계로는 아래 순서가 자연스럽습니다.

1. 정보구조와 화면 구조를 페이지 단위로 구체화한다.
2. DB 스키마를 설계한다.
3. 화면 와이어프레임과 컴포넌트 구조를 만든다.
4. 실제 프로젝트를 초기 세팅하고 구현을 시작한다.

## 8. 성공 기준

MVP의 성공 기준은 화려함이 아니라 아래 조건입니다.

- 유저가 3분 안에 가입하고 첫 운동을 시작할 수 있다.
- 운동 중에 기록 입력이 귀찮지 않다.
- 다음 운동 추천이 납득 가능하다.
- 대시보드에서 내 성장 추세가 한눈에 보인다.
- 친구/배지가 다시 접속할 이유를 만든다.

이 기준을 만족하면, 이후의 AI 고도화는 훨씬 쉬워집니다.
