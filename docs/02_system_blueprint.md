# STRONGER 시스템 청사진

## 1. 서비스 구조

서비스는 크게 3개 레이어로 나누는 것이 좋습니다.

### 1) 사용자 서비스 레이어
- 웹 앱
- 모바일 대응 UI
- 인증
- 대시보드
- 운동 기록
- 추천 보기
- 소셜 기능

### 2) 비즈니스 로직 레이어
- 운동 기록 처리
- 통계 집계
- 프로그램 규칙 엔진
- 추천 엔진
- 성취 계산
- 피드 생성
- 알림 처리

### 3) 운영 / 관리 레이어
- 관리자 페이지
- 회원 관리
- 신고 처리
- AI 품질 검수
- 랭킹 관리
- 운영 통계

- 연결 운동 목록

## 3. 데이터 흐름

### 운동 기록 저장 흐름
1. 사용자가 운동 시작
2. 세트 데이터를 입력
3. 세션 종료 시 서버 저장
4. 세션 완료 이벤트 발행
5. 분석 집계와 추천 엔진이 후속 처리

### 추천 생성 흐름
1. 최근 기록 조회
2. 프로그램 규칙 조회
3. 피로도 / 실패 여부 확인
4. 시간 제약 확인
5. 추천안 생성
6. 설명 문구 생성
7. 사용자에게 노출

### 분석 생성 흐름
1. 세션 완료 이벤트 수신
2. 총 볼륨 계산
3. 운동별 추세 계산
4. 주간 / 월간 집계 반영
5. 대시보드 카드 갱신

### 소셜 반영 흐름
1. PR 또는 세션 완료 이벤트 발생
2. 공개 설정 확인
3. 피드 아이템 생성
4. 친구 / 팔로워 타임라인 반영
5. 랭킹 / 성취 갱신

## 4. AI 추천 구조

첫 버전에서는 완전한 생성형 AI가 아니라, 아래 순서의 하이브리드 구조가 적합합니다.

### 1단계. 규칙 엔진
- 스트롱리프트, 웬들러 같은 방식별 규칙 적용
- 실패 누적 시 감량
- 피로 누적 시 세트 축소
- 운동 시간 부족 시 보조 운동 압축

### 2단계. 점수화 로직
- 최근 성공률
- 반복수 달성률
- 예상 1RM 변화
- 세션 회복 간격
- 루틴 이탈 빈도

### 3단계. 설명 레이어
- 왜 이 중량을 추천했는지
- 왜 운동 수를 줄였는지
- 왜 대체 운동을 제안했는지

이렇게 가야 추천의 신뢰성과 운영 가능성이 높습니다.
